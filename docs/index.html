Choose Your Installation Method

Pick the method that best fits your setup. Most users should start with Option A (NPM Install).

RECOMMENDED

Option A: NPM Install

Install the package globally on your computer. This is the recommended method for all users.

Step 1: Install globally

Open your terminal and run:

npm install -g @darbotlabs/darbot-browser-mcp

Windows users: You may need to run as Administrator. Mac/Linux users: You may need to prefix with sudo.

Step 2: Verify installation

Check that it installed correctly:

darbot-browser-mcp --version
Step 3: Update your MCP client config

When configuring your MCP client, use darbot-browser-mcp instead of npx:

{ "mcpServers": { "darbot-browser-mcp": { "command": "darbot-browser-mcp", "args": [] } } }
Step 4: Configure your MCP client

Jump to the MCP Client Configuration section below and follow the guide for your specific client (Claude Desktop, Claude Code, VS Code, etc.).

Option B: VS Code Extension

If you use VS Code with GitHub Copilot, this is the easiest method. The extension handles all configuration automatically.

Step 1: Install the extension

Click the button below or search for "Darbot Browser MCP" in VS Code Extensions:

Install in VS Code

Or install from command line:

code --install-extension darbotlabs.darbot-browser-mcp
Step 2: Restart VS Code

Close and reopen VS Code completely to activate the extension.

Step 3: Enable MCP (if prompted)

The extension may prompt you to enable MCP support. Click "Yes" to allow it to configure settings automatically.

Step 4: Start using

Open GitHub Copilot Chat (Ctrl+Shift+I) and try: @darbot-browser-mcp navigate to github.com

Option C: .NET SDK (For C# Developers)

Use Darbot Browser MCP directly in your .NET applications.

dotnet add package DarbotLabs.Browser.MCP

See the SDK & API Reference section for usage examples.

Quick Start

Your first 5 minutes with Darbot Browser MCP

âœ“ Before continuing, make sure you have:

  1. Completed the Installation steps above
  2. Configured your MCP client (Claude Desktop, Claude Code, VS Code, etc.) using the MCP Client Configuration guides
  3. Restarted your MCP client application

Step 1: Verify the Connection

First, let's make sure Darbot Browser MCP is connected and working.

In Claude Desktop:

Look for the plug icon or hammer icon in the message input area. Click it to see available tools. You should see "darbot-browser-mcp" listed with 52 tools.

In Claude Code (CLI):

Type /mcp to see connected servers:

/mcp

You should see "darbot-browser-mcp" in the list of active servers.

In VS Code with Copilot:

Open GitHub Copilot Chat panel. Type @ and you should see @darbot-browser-mcp as an option.

Not seeing it?
  • Make sure you completely restarted your MCP client (quit and reopen, don't just close the window)
  • Double-check your configuration file for typos (especially quotes and commas in JSON)
  • Check that Node.js is installed: run node --version in terminal
  • Try running the server manually to see errors: npx @darbotlabs/darbot-browser-mcp@latest

Step 2: Your First Browser Command

Let's navigate to a website. Simply ask your AI assistant in natural language:

Navigate to github.com

What happens:

  1. A browser window opens (you'll see it appear on your screen)
  2. The browser navigates to github.com
  3. The AI receives a "snapshot" of the page structure (accessibility tree)
  4. The AI can now interact with elements on the page
Pro Tip

The browser stays open between commands. You can have a conversation about what's on the page, and ask the AI to click buttons, fill forms, or navigate further.

Step 3: Try These Common Tasks

Copy and paste these examples to learn what's possible:

"Click the Sign In button"
"Click on the first search result"
"Scroll down to see more content"

"Type 'hello world' in the search box"
"Fill the email field with test@example.com"
"Select 'United States' from the country dropdown"
Session Management
"Save this browser session as 'my-research'"
"List all my saved browser sessions"
"Load the session called 'my-research'"
Multi-Tab Browsing
"Open a new tab and go to twitter.com"
"Show me all open tabs"
"Switch to the first tab"
Autonomous Crawling (Advanced)
"Start autonomous crawl of docs.python.org with max depth 2"

This will automatically explore a site, following links and taking screenshots. Great for research and documentation gathering.

Step 4: Closing the Browser

When you're done, you can close the browser:

"Close the browser"

Or just close your MCP client - the browser will close automatically.
