{+} Installation

Multiple ways to get started with Darbot Browser MCP

NPX (Recommended - Zero Install)

# Run directly with npx npx @darbotlabs/darbot-browser-mcp@latest # With Microsoft Edge npx @darbotlabs/darbot-browser-mcp@latest --browser msedge # Headless mode with SSE server npx @darbotlabs/darbot-browser-mcp@latest --headless --port 8931

VS Code Extension

# Install from command line code --install-extension darbotlabs.darbot-browser-mcp # Or search in VS Code marketplace: "Darbot Browser MCP"

Global NPM Install

npm install -g @darbotlabs/darbot-browser-mcp darbot-browser-mcp --browser msedge

.NET SDK

dotnet add package DarbotLabs.Browser.MCP

>_ Quick Start

VS Code + GitHub Copilot

// 1. Install the VS Code extension // 2. Enable MCP in settings.json: { "chat.mcp.enabled": true } // 3. Start chatting with Copilot: "Navigate to github.com and take a screenshot" "Fill out the login form with test@example.com" "Save this browser session as 'research-profile'"

MCP Configuration

// VS Code settings.json { "chat.mcp.enabled": true, "chat.mcp.servers": { "darbot-browser-mcp": { "command": "npx", "args": ["@darbotlabs/darbot-browser-mcp@latest"] } } }

<> VS Code Extension

Auto-Configuration

The extension automatically configures MCP settings when installed:

  • [+] Prompts to enable MCP in VS Code settings
  • [+] Configures the Darbot Browser MCP server
  • [+] Sets up optimal server path with npx @darbotlabs/darbot-browser-mcp@latest
  • [+] Adds status bar indicator for server health
